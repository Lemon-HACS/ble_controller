# Notify와 상태 확인

BLE Controller의 스위치 엔티티에서는 **Notify Characteristic**을 설정하여 디바이스의 실제 상태를 확인할 수 있습니다. 이 문서는 Notify의 동작 방식과 설정 방법을 설명합니다.

---

## Notify란?

BLE에서 **Notify**는 디바이스가 자신의 상태 변화를 연결된 클라이언트(Home Assistant)에게 능동적으로 알려주는 기능입니다.

일반적인 흐름:
```
HA → 디바이스:  "전원 켜" (Write)
디바이스 → HA:  "지금 켜진 상태야" (Notify)
```

Notify가 없으면 HA는 명령을 보낸 뒤 디바이스가 실제로 반응했는지 알 수 없습니다. Notify를 설정하면 디바이스의 실제 상태를 기반으로 엔티티 상태가 업데이트됩니다.

---

## 설정 항목

Notify 설정은 **선택 사항**입니다. 설정하지 않아도 스위치는 정상 동작하며, 명령을 보낸 즉시 상태가 변경된 것으로 간주합니다.

| 항목 | 설명 |
|------|------|
| **Notify Characteristic UUID** | 디바이스가 상태를 보내는 Characteristic UUID |
| **ON 응답 패턴 (hex)** | 디바이스가 ON 상태일 때 보내는 응답에 포함되는 hex 패턴 |
| **OFF 응답 패턴 (hex)** | 디바이스가 OFF 상태일 때 보내는 응답에 포함되는 hex 패턴 |
| **상태 조회 커맨드 (hex)** | 연결 직후 디바이스에 보내는 "현재 상태 알려줘" 명령 |

> **참고**: ON/OFF 응답 패턴이나 상태 조회 커맨드를 설정하려면 Notify UUID가 반드시 필요합니다. Notify UUID 없이 패턴만 입력하면 저장 시 에러가 발생합니다.

### 패턴 매칭 방식

ON/OFF 패턴은 **부분 일치**로 동작합니다. Notify 응답 데이터 안에 해당 패턴이 **포함**되어 있으면 매칭됩니다.

예시: 디바이스가 ON일 때 `0AA554050000640001001F`를 보내고, OFF일 때 `0AA5000A0000000000000000`를 보낸다면:
- ON 패턴: `0aa554` (응답의 앞부분)
- OFF 패턴: `0aa5000a` (응답의 앞부분)

전체 응답을 입력할 필요 없이, 상태를 구분할 수 있는 최소한의 패턴만 입력하면 됩니다.

---

## 상태 조회 커맨드

일부 디바이스는 "지금 상태가 뭐야?" 라는 질의 명령을 지원합니다. 이 명령을 보내면 디바이스가 Notify로 현재 상태를 응답합니다.

상태 조회 커맨드를 설정하면:
- **연결 직후**: 디바이스에 자동으로 상태 조회를 보내 현재 상태를 파악
- **Keep Alive 활성화 시**: 재연결할 때마다 자동 전송 + 주기적으로 상태 갱신

이를 통해 HA 재시작 후에도 스위치 상태가 "알 수 없음"이 아닌 실제 상태로 즉시 반영됩니다.

---

## Persistent Notify 구독

BLE Controller는 **Persistent Notify** 방식을 사용합니다. BLE 연결 시 1회 Notify를 구독하고, 연결이 유지되는 동안 구독을 유지합니다.

- 매 Write마다 Notify를 구독/해지하지 않아 오버헤드가 없음
- 디바이스가 언제든 보내는 Notify를 놓치지 않음
- 연결이 끊어지면 구독도 자동 해지되고, 재연결 시 다시 구독

---

## Keep Alive와 Notify

Keep Alive를 Notify와 함께 사용하면 가장 강력한 상태 동기화가 가능합니다.

1. HA 시작 → 디바이스 자동 연결 → Notify 구독 → 상태 조회 전송 → 현재 상태 반영
2. 주기적 keepalive ping → 상태 조회 재전송 → 상태 갱신
3. 연결 끊김 → 자동 재연결 → Notify 재구독 → 상태 조회 재전송

외부 요인(물리 버튼, 다른 앱)으로 디바이스 상태가 바뀌어도 Notify를 통해 HA에 자동 반영됩니다.

---

## Notify가 필요한 경우

- 디바이스가 외부 요인(물리 버튼, 다른 앱)으로 상태가 변경될 수 있는 경우
- HA 재시작 후 디바이스 상태를 즉시 파악해야 하는 경우
- 디바이스 상태를 정확하게 파악해야 하는 자동화가 있는 경우

## Notify가 불필요한 경우

- HA에서만 디바이스를 제어하고 상태 확인이 중요하지 않은 경우
- 디바이스가 Notify를 지원하지 않는 경우
- 단순 ON/OFF만 하고 상태 동기화가 필요 없는 경우
