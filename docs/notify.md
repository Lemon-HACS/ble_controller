# Notify와 상태 확인

BLE Controller의 스위치/셀렉트 엔티티에서는 **Notify Characteristic**을 설정하여 디바이스의 실제 상태를 확인할 수 있습니다. 이 문서는 Notify의 동작 방식과 설정 방법을 설명합니다.

---

## Notify란?

BLE에서 **Notify**는 디바이스가 자신의 상태 변화를 연결된 클라이언트(Home Assistant)에게 능동적으로 알려주는 기능입니다.

일반적인 흐름:
```
HA → 디바이스:  "전원 켜" (Write)
디바이스 → HA:  "지금 켜진 상태야" (Notify)
```

Notify가 없으면 HA는 명령을 보낸 뒤 디바이스가 실제로 반응했는지 알 수 없습니다. Notify를 설정하면 디바이스의 실제 상태를 기반으로 엔티티 상태가 업데이트됩니다.

---

## 설정 방법

Notify 설정은 **선택 사항**입니다. 설정하지 않아도 스위치/셀렉트는 정상 동작하며, 명령을 보낸 즉시 상태가 변경된 것으로 간주합니다.

### 스위치의 Notify 설정

| 항목 | 설명 |
|------|------|
| **Notify Characteristic UUID** | 디바이스가 상태를 보내는 Characteristic UUID |
| **ON 응답 패턴 (hex)** | 디바이스가 ON 상태일 때 보내는 응답의 hex 패턴 |
| **OFF 응답 패턴 (hex)** | 디바이스가 OFF 상태일 때 보내는 응답의 hex 패턴 |

예시: 디바이스가 ON일 때 `0AA554050000640001001F`, OFF일 때 `0AA5000A0000000000000000`을 보낸다면, 이 패턴을 입력하면 HA 스위치 상태가 실제 디바이스 상태와 동기화됩니다.

---

## Notify가 필요한 경우

- 디바이스가 외부 요인(물리 버튼, 다른 앱)으로 상태가 변경될 수 있는 경우
- 명령 전송과 실제 동작 사이에 시간차가 있는 경우
- 디바이스 상태를 정확하게 파악해야 하는 자동화가 있는 경우

## Notify가 불필요한 경우

- HA에서만 디바이스를 제어하고 상태 확인이 중요하지 않은 경우
- 디바이스가 Notify를 지원하지 않는 경우
- 단순 ON/OFF만 하고 상태 동기화가 필요 없는 경우
